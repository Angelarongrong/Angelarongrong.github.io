写在前面

5月20日，这是一个特别的周六（狗粮特别多？）——自己搭建的博客终于成功上传github啦（虽然现在看就是个特别朴素的原始部落嗡）。
第一次跟着老师和学长们做简单的编程项目（并希望能一步步被推向深坑）。在此之前，先搭建了个属于自己的blog。然后这是第一篇除了实验报告之外的技术性文档，大致地记录了一下这个blog的搭建过程。虽然个人觉得一路看教程、问搜索引擎、问学长而搭起来的blog并没有多少技术含量可言，但也算是一个小小的进步吧。
希望以后会是一个能自己解决问题的好孩子啊~

建站过程

安装所需应用程序及插件：

这里我参考的是官网的建站教程，里面提供了应用程序的下载地址。所需的应用程序有：

  • Node.js（据说安装后需要重启电脑，否则可能提示无效命令）
  • Git
    安装之后在任意地方右键会出现Git Bash选项，选择然后使用以下命令安装hexo：

$ npm install hexo-cli -g

安装插件：

npm install hexo-renderer-jade –save
npm install hexo-renderer-sass –save
npm install hexo-deployer-git –save

这一步在官网的教程中并未提到，但是却是不可或缺。这也正是我纠结了两个晚上愣是没法把blog上传成功的原因。讲道理，一定要安装啊，特别是第三个！各种各样的迷之错误已经让我无比绝望了QAQ

开始建站啦

首先要创建一个放置blog文件的文件夹。
在一个合适（喜欢）的位置创建一个文件夹，起一个好听（喜欢）的英文名。然后进入文件夹，右键选择Git Bash，执行下列命令：

$ hexo init
$ npm install

然后会发现这个文件夹里已经搭建好了blog所需的所有文件。
目录是这样的：

此时在文件夹内执行下列命令：

$ hexo g

其完整命令为hexo generate，用于生成静态文件

$ hexo s

其完整命令为hexo server，用于启动服务器，主要用于本地预览
此时用浏览器访问http://localhost:4000/，可以本地预览到blog啦~

####部署到Github上

先生成静态文件

$ hexo g

$ hexo d

hexo d完整命令为hexo deploy，用于将本地文件发布到github上。
命令成功执行之后可以在仓库中看到上传的所有blog的文件。大约几分钟后可以在浏览器访问github用户名.github.io，无误的话能看到自己的博客。

这一步还有另一种做法，需要配置SSH，由于操作时（可能是插件原因）出错了，没有最终完成（但大部分操作都成功了）在技术参考的第三个链接里有十分详细的操作过程。个人整理如下：

配置代码：
type: git
repository: git@github.com:Angelarongrong/Angelarongrong.github.io.git
branch: master

配置SSH：
$ ssh-keygen -t rsa -C “Github注册时的邮箱”

之后会要求确认路径与输入密码，可默认一路回车。然后在C:\Users\Administrator下若找到.ssh文件夹则表示成功了。以文本形式打开文件夹中的id_rsa.pub，复制里面的全部内容，回到github页面，点击右上角图标在下拉菜单中选择Settings>SSH and GPG Keys>New SSH Key.title任意填写，将复制的内容粘贴到key中保存。为了验证是否成功，在git bash中输入：

$ ssh -T git@github.com

第一次输入会有提示，输入yes如果看到以下内容则成功连上了github：

在上传本地文件之前还需要设置username和email：

$ git config –global user.name “github用户名”

$ git config –global user.email “github注册邮箱”

（其实并不知道这两步有没有起作用）
接着有如下步骤（中间按回车就可以），据说设置完之后使用hexo d命令发布内容就不需要输入密码了

注册Github账号

用户名、邮箱、密码需要做好记录。

创建repository

这是一个相当于仓库的、用于放置blog文件的地方。登录进入Github后想方设法找到一个new repositories的按钮。比如可以这样：

选择start a project
进入这样子的页面

只需要填写repository name，然后须十分注意它的格式：

github用户名.github.io

我的用户名是Angelarongrong，所以我的仓库名叫Angelarongrong.github.io
点create就完成啦

新建博客文章

不知道是不是一定要在source>_posts中右键git bash才能成功执行命令，总之在别的地方git bash，.md文件都没有成功生成。如果有人看这篇东西，恰好遇到同样问题，不妨一试。

$ hexo new “my new post”

随后用记事本或者npp在文件夹中打开生成的文件。正文使用markdown语法格式书写。
这是我十分伪抒情的第一篇文章。

完成后用$ hexo g生成静态文件，用hexo s在本地预览效果。感觉没问题了，用hexo d同步到github。就可以用浏览器访问啦~
（中途get许多markdown小语法）
（马克飞象界面真的很美好吗）

删除文章：

  • 在_posts文件夹中先将源文件删除
  • 执行命令：$ hexo clean (清除缓存db.json和已经生成的静态文件public)
  • 执行命令： $ hexo g
    刷新页面~
